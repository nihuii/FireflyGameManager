现在游戏封面成功的导入进来了，但是将bangumi的所有字段全部导入后游戏详情页游戏资料依然为空，点击来源浏览器跳转到https://bgm.tv/subject/172612，但是提示“数据库中没有查询您所指定的条目”，点击刷新提示“在线条目不存在或暂时不可用”



**核心原因**

这不是保存失败，也不是 UI 没刷新，而是 Bangumi 搜索接口和详情接口返回了互相矛盾的数据：

1. 旧搜索接口仍能搜到 `172612`，并返回名称、封面。
2. 当前 V0 详情接口对同一 ID 返回 `404`：[详情接口](https://api.bgm.tv/v0/subjects/172612)。
3. 网页端同样把它显示成不存在：[条目页面](https://bgm.tv/subject/172612)。
4. 该条目此前的 API 响应标记为 `nsfw=true`，而你的本地 Bangumi 账号目前是 `RequiresReconnect=true`。

因此，`172612` 目前相当于一个“搜索可见、详情不可见”的幽灵结果。可能是受限条目的认证策略，也可能是 Bangumi 新旧接口之间的数据不一致。我们实际观察到同一个未认证接口先返回过完整资料，随后又变成 404，说明服务器侧行为并不稳定。

**为什么封面能导入**

搜索接口返回的数据里有：

```
名称：千恋＊万花
Subject ID：172612
封面：https://lain.bgm.tv/...
```

封面下载是独立流程，不需要详情接口。因此应用能够下载图片、写入封面缓存并更新游戏库。

**为什么勾选全部字段仍然为空**

“勾选全部字段”只代表：

> 把资料预览中已有的字段复制到本地。

它不会额外生成或推断资料。详情接口返回 404 后，应用只能使用搜索结果创建降级预览。而搜索结果中实际是：

```
名称：千恋＊万花
封面：有
简介：空
发行日期：空
开发商：空
发行商：空
标签：[]
```

所以即使全部勾选，复制到数据库里的仍然是空值。当前 SQLite 记录也验证了这一点：

```
subject_id    172612
is_linked     1
summary_len   0
release_date  空
developer     空
publisher     空
tags_json     []
image_url     有
```

**为什么刷新失败**

详情页刷新仍然调用：

```
GET https://api.bgm.tv/v0/subjects/172612
```

代码现在会重试一次，但服务器连续返回两次 404，最终只能提示“在线条目不存在或暂时不可用”。重试只能解决瞬时故障，不能解决受限访问或新旧接口数据不一致。

**为什么来源页面也不存在**

来源按钮只是打开：

```
https://bgm.tv/subject/172612
```

浏览器不会自动携带 FireflyGameManager 保存的 Access Token。若网页要求 Bangumi 登录、条目受访问限制，或条目已从新版数据库隐藏，浏览器就会显示“数据库中没有查询您所指定的条目”。

**账号目前也没有参与资料读取**

项目当前的资料详情调用是匿名请求。虽然应用有 Bangumi 账号存储，但 `BangumiGameMetadataProvider` 获取游戏详情时没有把 Access Token 传给 API；同时你保存的账号状态已经是 `RequiresReconnect=true`。

因此，单纯在现有版本中重新连接账号也还不足以解决问题，代码需要同时改成：

1. 有有效账号时，使用 Access Token 请求受限条目的完整详情。
2. 匿名详情返回 404 时，改用旧搜索接口的 `large` 数据补充简介和发行日期。
3. 无法获取开发商、发行商、标签时，明确显示“部分资料”，不再让“全部导入”产生误导。
4. 区分“条目真的不存在”和“可能需要登录或重新连接账号”。





**方案 A：完整修复，推荐**

1. **让资料请求使用 Bangumi 账号**
   - 账号有效时，将 Access Token 加入详情请求。
   - 受限条目若允许该账号访问，就能获取简介、日期、开发商、发行商和标签。
   - 401/403 时自动标记为需要重新连接，不泄露 Token。
2. **匿名状态使用旧搜索接口降级**
   - 将旧搜索改为 `responseGroup=large`。
   - 即使详情 API 返回 404，仍可导入名称、封面、完整简介和发行日期。
   - 明确显示“部分资料”，不再把空字段说成全部导入成功。
3. **详情页支持自愈刷新**
   - 当前 `172612` 刷新失败时，根据本地游戏名重新搜索。
   - 找到相同 subject ID 后，用搜索结果补齐简介和日期。
   - 账号重连后再次刷新，可进一步补齐开发商、发行商和标签。
4. **区分三种错误**
   - 未登录或账号失效：提示重新连接。
   - 受限条目匿名不可见：提示当前仅能导入部分资料。
   - 搜索和详情均不存在：才提示条目已删除。
5. **来源按钮**
   - 保留条目地址，但提示网页端也可能要求单独登录 Bangumi。
   - 应用 Token 无法自动登录外部浏览器，这是两个独立登录环境。
